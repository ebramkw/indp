Detailed Description

This application runs Chaos periodically. A round is started by one node (initiator), the application prints statistics at the end of the round.

The application schedules Chaos periodically with a fixed period CHAOS_PERIOD.

The duration of each Chaos phase is given by CHAOS_DURATION.

During each Chaos phase, the maximum number of transmissions in Chaos (N) is set to N_TX.

The initiator of the floods is the node having nodeId INITIATOR_NODE_ID.

The packet to be flooded has the format specified by data structure chaos_data_struct.

Receivers synchronize by computing the reference time during each Chaos phase.

To synchronize fast, at startup receivers run Chaos with a significantly shorter period (CHAOS_INIT_PERIOD) and longer duration (CHAOS_INIT_DURATION).

Receivers exit the bootstrapping phase when they have computed the reference time for CHAOS_BOOTSTRAP_PERIODS consecutive Chaos phases.
